Objetivo: Garantizar la integridad de la experiencia del usuario y la gestión de la latencia.


Contexto: UI para asesores senior que requiere feedback inmediato para evitar dobles clics.



Reglas técnicas obligatorias:


Feedback de Latencia: Toda acción asíncrona > 1s MUST mostrar un Skeleton Screen.



Optimistic UI: El mensaje del usuario se muestra instantáneamente en el chat antes de la confirmación del servidor.






Manejo de Errores: Errores de red MUST disparar un "Retry" con la misma Idempotency-Key.



Validaciones requeridas: El session_id de la URL debe ser un UUID v4 válido; si es inválido, redirigir a una nueva sesión.
