Objetivo: Definir los boundaries y la visión sistémica del proyecto Travel-Mind.


Contexto: Arquitectura de "Nodo Soberano" donde el Frontend gestiona la verdad operativa y el Backend la persistencia inmutable.

Alcance permitido: Despliegue en Cloud Run, Firestore Native y Vertex AI Search.

Reglas técnicas obligatorias:


BFF (Backend for Frontend): FastAPI MUST actuar como único punto de entrada, abstrayendo Vertex AI Search y Firestore.



Resource-Oriented Design (ROD): Las APIs deben estructurarse en torno a los recursos /sessions y /messages.





Inmutabilidad: Los mensajes en Firestore son append-only; no se permite el método PATCH para el texto del mensaje.


Condiciones de parada: Detener ejecución si no se confirma la conectividad con el Data Store de Vertex AI
